2. ESTRUTURA DO BANCO DE DADOS (SQLite)

2.1 Tabela: profissionais

CREATE TABLE profissionais (
    id VARCHAR(20) PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    especialidade VARCHAR(50),
    preco_sessao DECIMAL(10,2),
    chave_pix VARCHAR(100),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

2.2 Tabela: pacientes

CREATE TABLE pacientes (
    id VARCHAR(20) PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    telefone VARCHAR(20),
    data_nasc DATE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

2.3 Tabela: sessoes

CREATE TABLE sessoes (
    id VARCHAR(20) PRIMARY KEY,
    profissional_id VARCHAR(20) NOT NULL,
    paciente_id VARCHAR(20) NOT NULL,
    data DATETIME NOT NULL,
    status VARCHAR(20) DEFAULT 'agendada',
    valor DECIMAL(10,2),
    pagamento_status VARCHAR(20) DEFAULT 'pendente',
    FOREIGN KEY (profissional_id) REFERENCES profissionais(id),
    FOREIGN KEY (paciente_id) REFERENCES pacientes(id)
);

2.4 Tabela: pagamentos

CREATE TABLE pagamentos (
    id VARCHAR(20) PRIMARY KEY,
    sessao_id VARCHAR(20) NOT NULL,
    valor DECIMAL(10,2) NOT NULL,
    gateway VARCHAR(20) NOT NULL,
    status VARCHAR(20) NOT NULL,
    transacao_id VARCHAR(100),
    data DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (sessao_id) REFERENCES sessoes(id)
);

5. SINCronização com DROPBOX

5.1 Estrutura de Pastas

/IntegrativoApp-Demo/
├── database.db           # Banco de dados principal
├── logs/
│   └── sync_log.json     # Log de sincronização
├── backups/
│   └── backup_YYYYMMDD.db
└── config/
    └── ambiente_teste.json

5.2 Frequência de Sincronização

  • Automática: A cada 5 minutos ou após cada transação
  • Manual: Botão "Sincronizar Agora" na interface
  • Recuperação: Últimos 5 backups mantidos
