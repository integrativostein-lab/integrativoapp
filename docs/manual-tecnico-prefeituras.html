2. Download e Instalação

2.1. Obter o código-fonte

Acesse a página de download com a chave de ativação fornecida pelo Integrativo.App:

https://integrativo.app/ativacao?chave=SUA_CHAVE_AQUI

Faça o download do arquivo integrativo-entidade.zip e extraia no servidor.

2.2. Instalar dependências

cd /opt/integrativo/backend
npm install

2.3. Configurar o banco de dados

Crie um banco PostgreSQL e configure o arquivo .env:

DATABASE_URL=postgresql://usuario:senha@localhost:5432/integrativo_entidade
JWT_SECRET=chave_secreta_gerada_pela_entidade
FRONTEND_URL=https://saude.SUA_ENTIDADE.org.br

2.4. Criar as tabelas

node database.js

2.5. Iniciar o servidor

node server.js

O sistema estará disponível em http://localhost:3000.

💡 Para produção, utilize um gerenciador de processos como PM2: npm install -g pm2 && pm2 start server.js --name integrativo

3. Configuração do Domínio e HTTPS

Recomendamos usar Nginx como proxy reverso com certificado SSL via Certbot (Let's Encrypt).

server {
    listen 443 ssl;
    server_name saude.SUA_ENTIDADE.org.br;

    ssl_certificate /etc/letsencrypt/live/saude.SUA_ENTIDADE.org.br/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/saude.SUA_ENTIDADE.org.br/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

4. Modo de Homologação

⚠️ Após a instalação, o sistema opera em Modo de Homologação com as seguintes restrições:
  • Marca d'água "AMBIENTE DE HOMOLOGAÇÃO" em todas as telas
  • Nota Fiscal bloqueada (documentos gerados com tarja "Inválida")
  • Exportação DATASUS desabilitada
  • Assinatura digital bloqueada
Para liberar o Modo Produção, o responsável legal da organização social deve assinar o Termo de Parceria Social com certificado digital ICP-Brasil (e-CNPJ) e inseri-lo no painel de administração.

5. Integração com o DATASUS (BPA)

O sistema gera arquivos no formato BPA (Boletim de Produção Ambulatorial) compatíveis com o DATASUS.

5.1. Configurar CNES e CNS

Cada profissional deve ter seu número CNES cadastrado. Os pacientes devem ter o CNS (Cartão Nacional de Saúde) informado no cadastro.

5.2. Gerar arquivo de remessa

No painel de administração: Relatórios → Exportar DATASUS. O sistema gera um arquivo .txt pronto para envio.

5.3. Envio automático

Configure o endpoint do WebService do DATASUS em Configurações → Integrações → SUS. O sistema pode enviar automaticamente os arquivos gerados.

6. Backup e Restauração

6.1. Backup automático

O sistema realiza backup automático semanal (diário para planos pagos). Os arquivos são salvos em /opt/integrativo/backups/.

6.2. Restauração manual

psql -U usuario -d integrativo_entidade < backup_2026-06-15.sql

7. Atualização do Sistema

Para atualizar para uma nova versão:

cd /opt/integrativo
git pull origin main
cd backend
npm install
node database.js
pm2 restart integrativo
